基于注解驱动的全栈开发框架，通过丰富的注解体系实现前后端强映射关系，简化企业级应用开发

快速开始 查看文档

核心特性

Ooder 框架提供完整的全栈开发解决方案

注解驱动开发

通过声明式注解配置实现视图结构和行为定义，将配置与业务逻辑分离，提高代码可维护性

组件化设计

采用"干-支-叶"结构化设计模式，将复杂功能拆分为可复用的组件模块，支持复杂业务场景

全栈架构

后端注解与前端映射无缝集成，提供统一的技术栈和开发体验，降低技术复杂度

多主题支持

内置亮色、暗色、高对比度等多种主题，支持自定义主题扩展，满足不同用户需求

响应式设计

完美适配桌面、平板、手机等多种设备，提供一致的用户体验

企业级特性

提供权限管理、数据验证、国际化等企业级功能，支持大规模应用开发

快速开始

几分钟内开始使用 Ooder 框架

1. 安装依赖

在 Maven 项目中添加依赖：

<dependency> <groupId>net.ooder</groupId> <artifactId>ooder-annotation</artifactId> <version>1.0.1</version> </dependency>

2. 创建业务模块

使用注解定义业务模块：

@BusinessModule("示例模块") public class ExampleModule { // 业务逻辑实现 }

3. 配置前端映射

在 resource/static/ood 目录下配置前端资源：

// 前端组件配置 // 自动映射后端注解
