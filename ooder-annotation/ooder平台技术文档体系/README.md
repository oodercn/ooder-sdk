# ooder平台技术文档体系

## 概述

本目录按照技术文档结构化规范对ooder平台的技术文档进行重新分类和组织，便于知识体系的构建和维护。

## 目录结构

### 1. 架构与核心概念
包含平台整体架构、核心设计理念、基础概念等内容

### 2. 注解体系
包含注解驱动框架的完整注解体系说明

### 3. 视图设计
包含各类视图组件的设计规范、使用方法和最佳实践

### 4. 服务架构
包含服务设计规范、服务类型定义、参数设计等内容

### 5. 通讯组件
包含各类通讯组件的使用规范和实现机制

### 6. 参数与上下文
包含参数设计规范、上下文环境管理等内容

### 7. 组件专题
包含特定组件的专题手册和深入分析
- Bar设计体系
- Combo组件
- EChart与FChart图表
- SVG视图与动画
- **CSS样式注解（版本2.0新增）**

### 8. 最佳实践
包含开发经验总结、最佳实践指南等内容

## 使用说明

1. 建议按照学习路径顺序阅读文档
2. 各目录下的文档保持相对独立，可根据需要选择性阅读
3. 定期更新文档内容，保持与平台发展的同步

## 维护规范

1. 新增文档应按照分类放置到相应目录
2. 过时或重复的文档应及时归档到archived_docs目录
3. 保持文档结构清晰，避免交叉引用混乱
